# Leetcode

# T79. 单词搜索

middle

给定一个 m x n 二维字符网格 board 和一个字符串单词 word 。如果 word 存在于网格中，返回 true ；否则，返回 false 。

单词必须按照字母顺序，通过相邻的单元格内的字母构成，其中 “相邻” 单元格是那些水平相邻或垂直相邻的单元格。同一个单元格内的字母不允许被重复使用。

示例

示例 1：

输入：board = [["A","B","C","E"],["S","F","C","S"],["A","D","E","E"]], word = "ABCCED"
输出：true

示例 2：

输入：board = [["A","B","C","E"],["S","F","C","S"],["A","D","E","E"]], word = "SEE"
输出：true

示例 3：

输入：board = [["A","B","C","E"],["S","F","C","S"],["A","D","E","E"]], word = "ABCB"
输出：false

提示：

  • m == board.length
  • n = board[i].length
  • 1 <= m, n <= 6
  • 1 <= word.length <= 15
  • board 和 word 仅由大小写英文字母组成

进阶：

你可以使用搜索剪枝的技术来优化解决方案，使其在 board 更大的情况下可以更快解决问题

# 关键点

  1. 注意从哪回溯到哪，即应该撤销的操作到底是什么

    • 比如力扣第 79 题单词搜索：

    • 只要遍历到某结点，就设置该结点对应字符为 '.' ，然后用 backtracking() 遍历其所有子结点

    • ❗️调用递归函数后的回溯如图所示，撤消操作即：把原先的字符归还给函数指向的结点

  2. 注意回溯时无需手动将函数指回父结点

    • 因为递归函数中的 return 就自动帮我们回到上层了

    • 比如力扣第 79 题单词搜索：

      回溯时不需要把结点对应棋盘上的点坐标回溯，因为 i j 是递归函数的参数，所以坐标会随着递归函数的 return 而回溯（意会，可结合组合问题的案例思考，该案例的起始坐标 i 就没有手动地减回去）

  3. 条件”||“的判断顺序很重要

    比如力扣第 79 题单词搜索：

    java.lang.AnyindexOutOfBoundsException: Indext -1 out of length 3

    原因：从前往后判断， (i, j) 可能会有不在棋盘边界内的情况

    • 解决：将第⼀个判断条件挪至最后

    • 解释：这样会先判断坐标是否在棋盘边界内，由 || 运算符的特性，前面有值为 1 的情况，就不再向后判断，所以若 (i, j) 出了边界，就不会进行到 board[i][j] 这步
