# 带符号数的表示

  • 真值

    📗==+0.1011、-==0.1011

  • 机器数

    在计算机内部使用，将符号数值化

    由符号位 + 有效位构成

    有原码、反码、补码表示

    • 原码

      📗若 X=+0.6875，则 X=+0.1011000B；X原_原原​ =0.1011000

      真值 0 有两种不同表示：

      • [+0]原_原原​ =00000000B
      • [-0]原_原原​ =10000000B

      n 位整数原码（含一位符号）表示范围：-(2n−1^{n-1}n−1-1) ~ +(2n−1^{n-1}n−1-1)

      n 位纯小数原码（含一位符号）表示范围：-(1-2−(n−1)^{-(n-1)}−(n−1)) ~ +(1-2−(n−1)^{-(n-1)}−(n−1))

    • 反码

      正数：与原码同

      负数：有效位为真值有效位各位取反

      📗若 X=+0.1011B，则 X反_反反​ =0.1011；若 X=-0.1011B，则 X反_反反​ =1.0100

      真值 0 有两种不同表示：

      • [+0]反_反反​ =00000000B
      • [-0]反_反反​ =11111111B

      n 位整数反码（含一位符号）表示范围：-(2n−1^{n-1}n−1-1) ~ +(2n−1^{n-1}n−1-1)

      n 位纯小数反码（含一位符号）表示范围：-(1-2−(n−1)^{-(n-1)}−(n−1)) ~ +(1-2−(n−1)^{-(n-1)}−(n−1))

    • 补码

      • 正数：与原码同

        负数：有效位为真值有效位各位取反后末位加 1

      • 真值 0 只有一种表示：[+0]补=[−0]补_补=[-0]_补补​=[−0]补​ = 00000000B

      • 补码 10000000B 真值为 - 128（8 位补码表示的最小负数）

      • n 位整数补码（含一位符号）表示范围：-2n−1^{n-1}n−1~ +(2n−1^{n-1}n−1-1)

        n 位纯小数补码（含一位符号）表示范围：-1 ~ +(1-2−(n−1)^{-(n-1)}−(n−1))

    原码两次取补还是等于原码

  • 补码加减运算

    符号位一起参与运算

    • 补码加：(X+Y)补=X补+Y补_补 = X_补+Y_补补​=X补​+Y补​

    • 补码减：(X-Y)补=X补−Y补=X补+(−Y)补_补 = X_补-Y_补 = X_补+(-Y)_补补​=X补​−Y补​=X补​+(−Y)补​

      (-Y)补_补补​：将 Y补_补补​连同符号位一起各位变反，末位加 1，称为变补操作

# 浮点数的表示

  1. 浮点数的真值

    N = ± RER^ERE × M

    • N：真值
    • M：尾数
    • RER^ERE：比例因子
    • E：阶码
    • R：阶码的底，一般与尾数 M 的基数相同

  2. 浮点数的机器格式

    阶码 E：带符号定点整数，可用补码或移码表示。指明小数点位置，决定浮点数表示范围

    尾数 M：带符号定点小数，可用补码或移码表示。给出有效值位数，决定浮点数的精度

    尾数规格化：绝对值为 0.1 几。📗0.1001原_原原​，1.1001原_原原​，0.1001补_补补​，1.0111补_补补​，1.1补_补补​

  3. 移码（增码）

    常用于表示浮点数的阶码

    定义：真值 X 的 m+1 位移码表示为：X移=2m+XX_移=2^m+XX移​=2m+X

    📗若真值 X=-1011，则用 5 位移码表示为：X移=24+X=10000−1011=00101X_移 = 2^4+X = 10000-1011 = 00101X移​=24+X=10000−1011=00101

    移码的特点：

    1. 最高位为符号位，正数移码符号位为 1，负数移码符号位为 0
    2. 除符号位外，移码的其余各位与补码相同
    3. 随着真值 X 从最小增至最大，响应 X移_移移​从 00・・・00 增至 11・・・11，能更直观比较阶码大小

  4. IEEE754 标准浮点格式

    有 3 种浮点表示格式：短浮点数 32 位、长浮点数 64 位、临时浮点数 80 位

    类型数符（位）阶码（位）尾数数值（位）总位数（位）偏置值（十进制）
    短浮点数182332127
    长浮点数11152641023
    临时浮点数115648016383

# 第二章 指令系统

指令的执行可包含如下步骤：

  1. 取指令：从主存中取指到 CPU
  2. 执行指令：在 CPU 内部执行指令（此过程中，CPU 可能还需从主存储器中获取操作数）
  3. 存储结果：将执行结果送回存储器

指令系统（= 指令集）：一台计算机所能执行的全部指令的集合

指令系统是计算机软硬件之间的界面

# 寻址方式

寻址方式：规定如何解释地址码，以找到操作数

可分为四大类，其他的寻址方式是他们的变型或组合

  1. 立即寻址
  2. 直接寻址类
  3. 间接寻址类
  4. 变址类

  • 立即寻址方式

    指令中直接给出操作数

    📗：MOV AX, 1234H

  • 直接寻址方式（绝对地址）

    指令给出操作数地址（存储单元号或寄存器号）

    1. （主存）直接寻址方式

      操作数 S 与地址码 A 的关系：S=(A)

      📗：MOV AX, [2000H]

    2. 寄存器寻址方式

      例如，模型机 CPU 的可编址寄存器编号可分配如下：

      R0=000,R1=001,R2=010,R3=011,R_0=000, R_1=001, R_2=010, R_3=011,R0​=000,R1​=001,R2​=010,R3​=011, SP=100, PSW=101, PC=11 等

      操作数 S 与寄存器RiR_iRi​ 的关系：S=(Ri_ii​)

      📗ADD R0,R3R_0, R_3R0​,R3​

      📗MOV AX, BX

  • 间接寻址

    间接寻址类的操作数都存在主存储器中

    1. 主存间接寻址方式

      地址码给的是间接地址

      操作数 S 与地址 A1 的关系：S=((A1))

    2. 寄存器间接寻址方式

      地址码给的是寄存器编号，寄存器中存放操作数地址

      操作数 S 与寄存器RiR_iRi​ 的关系：S=((Ri_ii​))

      📗：MOV AX, [BX] ; MOV AX, DS: [BX]

    3. 自增型寄存器间址方式

      寄存器间址的一种变型

      按照寄存器间址方式获取操作数后，寄存器里存的内容加 1

      操作数 S 与寄存器RiR_iRi​ 的关系：S=((Ri_ii​))

    4. 自减型寄存器间址方式

      寄存器间址的又一种变型

      寄存器里存的内容减 1，然后按照寄存器间址方式获取操作数

      操作数 S 与寄存器RiR_iRi​ 的关系：S=((Ri_ii​)-1)

    5. 堆栈寻址

      堆栈是一段特殊的存储空间，以向上生成（从高地址向低地址方向存数据）方式为例：

      1. 压栈：S=((SP)-1)

        压栈操作是 SP 作间址寄存器的自减型寄存器间址

      2. 出栈：S=((SP))

        得到操作数后 SP 里存的地址要加 1

        出栈操作是 SP 作间址寄存器的自增型寄存器间址

  • 变址、基址寻址

    通过计算得到有效地址，使地址表示灵活可变

    1. 变址寻址：

      指令中一个地址码包含两个信息：形式地址 D，变址寄存器 Rx_xx​

      操作数 S 与形式地址 D、变址寄存器 Rx 的关系：S=((Rx_xx​)+ D)

      📗：下列语句的源操作数采用变址寻址

      在程序执行过程中RxR_xRx​ 存的内容可变，D 不可变

    2. 基址寻址：

      指令中一个地址码给出了两个信息：形式地址 D，基址寄存器 RB_BB​

      操作数 S 与形式地址 D、基址寄存器 RB_BB​ 的关系：S=((RB_BB​)+ D)

      在程序执行过程中 RB_BB​ 内容不变，D 可变

    3. 基址加变址方式

      指令基本格式：OP | RB_BB​ | Rx_xx​ | D

      操作数的有效地址 = 基址寄存器内容 + 变址寄存器内容 + 形式地址

      ​ = 基准地址 + 变址偏移量 + 常规偏移量

      📗某商场的销售金额汇总如表所示，采用基址加变址方式查询某天的销售额。假定：存储首址为 1000H（内容为 1 月 1 日销售金额），每天的销售金额存放在一个主存单元之中，为每个月份分配 31 个单元

# 组合逻辑控制器概述

  1. 定义：每种微命令都需要一组逻辑电路，全机所有微命令所需的逻辑电路就构成了微命令发生器

# 组合逻辑控制器时序系统

模型机的组合逻辑控制器采用三级时序：工作周期（机器周期）、时钟周期（节拍）、工作脉冲

依靠不同的时间标志，使 CPU 分步工作

组合逻辑控制器的三级时序：

工作周期：不同工作周期长度可能不同

时钟周期（节拍）：时序系统中最基本的时间分段。各节拍长度相同

工作脉冲（定时脉冲）：有的操作如打入 R，需严格的时钟脉冲，以确定在哪一刻打入

  1. 工作周期

    • 指令正常执行过程的工作周期
      • 取指周期 (FT)
      • 源周期 (ST)
      • 目的周期 (DT)
      • 执行周期 (ET)
    • I/O 传送控制的工作周期
      • DMA 周期 (DMAT)
      • 中断周期 (IT)

    1. 取指周期 FT

      所有指令的公共操作。从存储器取指送 IR，修改 PC

    2. 源周期 ST

      源操作数非寄存器寻址，进入 ST。按寻址方式形成源操作数地址，从主存读数送 C

    3. 目的周期 DT

      目的操作数非寄存器寻址，则进入 DT。按寻址方式形成目的地址，非传送类指令还需从主存读数送 D

    4. 执行周期 ET

      根据操作码执行具体操作，并将后续指令地址（PC 的值）送 MAR

    5. DMA 周期 DMAT

      响应 DMA 请求之后进入 DMAT。实现主存与外围设备间的数据直传

    6. 中断周期 IT

      响应中断请求后需要一个过渡周期，为进入中断服务程序做准备，这个过渡周期称为中断周期 IT

    工作周期转换示意图：

    ❓指令执行过程如何控制工作周期的切换？

    ✅每个工作周期对应一个状态触发器

    • 0 -> 1 工作周期开始
    • 1 -> 0 工作周期结束

    采用集成 D 触发器

    同一时刻，只能有一个工作周期状态触发器为 1

  2. 时钟周期（节拍）

    1. 时钟周期长度

      一次访存时间作为时钟周期长度，设置为 1μs，可执行以下操作之一：

      • 一次读主存，并经内部数据通路传送
      • 一次内部数据通路传送
      • 一次写主存 (MDR -> M)

    2. 时钟周期计数

      用计数器 T 进行计数

      T：

      • 每个工作周期第一节拍，T=0
      • 每开始一个新节拍，T 计数（T+1)
      • 工作周期结束时，T 清 0

  3. 工作脉冲 P

    每个时钟周期末尾设置一个工作脉冲，作为各种同步脉冲的来源

    同步定时脉冲控制定时操作：打入寄存器、时序切换

      • NJP：转移条件不满足，不转移，顺序执行
        • PC 型：转移地址在当前指令下一单元
        • ¯PC 型：转移地址不在当前指令下一单元

    • RST，获取返回地址（来自栈顶），安排在 ET 中

    📗1：JMP R0;

    FT0: M -> IR, PC+1 -> PC

    ET0: R0 -> PC、MAR

    📗2：RST (SP)+;

    FT0: M -> IR, PC+1 -> PC

    ET0: SP ->MAR

    ET1: M -> MDR -> PC、MAR

    ET2：SP+1 -> SP

# 组合逻辑控制方式的优缺点

优点：0

控制信号产生速度快

缺点：

  1. 设计不规整：实现各微命令的门电路所需逻辑形态不规整，核心电路繁琐、零乱，设计效率较低
  2. 不易修改或扩展

# 模型机的微程序控制器

  1. 基本概念

    1. 微操作：计算机工作过程电路实现的最基本的操作，如：打开 / 关闭门电路等
    2. 微命令：构成控制信号序列的最小单位，用于控制微操作
    3. 微指令：一条微指令对应一个时钟周期，由多条微命令组成
    4. 微程序：一条微程序对应一个机器周期，由多条微指令组成
    5. 微周期：读取一条微指令并执行相应的异步操作所需的时间，通常是一个时钟周期

  2. 微程序控制器组成

    1. 微程序控制器的硬件组成

      1. 控制存储器 CM：存放微程序，每一单元存放一条微指令
      2. 微指令寄存器 μIR：用于存放从 CM 读取的微指令
        • 微操作控制字段（微命令字段）：产生微命令的依据（用于控制一步操作）
        • 顺序控制字段（微地址字段）：产生后续微地址的依据，用以控制微程序的连续执行
      3. 微地址形成电路：生成下一条要执行的微指令地址
      4. 微地址寄存器 μAR：存放下一条微指令地址，用于从 CM 读取微指令

    2. 时序系统

      同步控制，用统一微指令周期控制各条微指令执行

    3. 微程序控制方式的优缺点

      优点：

      1. 设计规整，设计效率高
      2. 易于修改、扩展指令系统功能

      缺点：

      1. 速度慢（访存频繁、转移较多）
      2. 执行效率不高

      应用范围：速度要求不高、功能较复杂
